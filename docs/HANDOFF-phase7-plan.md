## 개선 과제 — 실행 계획

### 1차: 즉시 효과 (~30분)

#### [A] 소스 다양성 개선 — 코드 3줄

- **현상**: 31건 중 S7(국내매체) 30건, 영어권 전문매체(S1/S2/S5) 0건
- **원인**: 영어 기사 `relevance_score`가 0.1 미만 → 임계값에서 전부 탈락
- **수정**:
  - `relevance_score` 임계값 0.1 → 0.05
  - `MINORITY_MIN` 2 → 3
  - `MAX_SOURCE_GROUP_RATIO` 0.4 → 0.35
- **기대효과**: S1~S5 기사 최소 3건씩 포함 → 글로벌 관점 확보

#### [C] 토큰 비용 절약 (무위험) — 20분

- Stage 4 (SK렌즈) Opus → Sonnet: 입력 625자, 품질 차이 없음 (~15% 절약)
- Stage 1 (클러스터링) Sonnet → Haiku: 분류 작업 (~5% 절약)
- Anthropic prompt caching 활성화 (~15~20% 절약)
- **합계**: 비용 35~40% 절감 ($7.5 → ~$4.5/월), 품질 영향 없음

---

### 2차: 핵심 품질 (~2.5시간)

#### [디자인] HTML/CSS 세련화 + 텍스트 버전 배포

- **현상**: MVP 수준 디자인 + MD 텍스트 브리핑이 웹에서 접근 불가
- **범위**:
  - 인덱스 + 브리핑 페이지 CSS/레이아웃을 Google Opal 수준으로 리디자인
  - MD 브리핑을 텍스트 HTML 페이지로 변환하여 `web/`에 배포
  - nav 바 + 인덱스에 "텍스트 버전" 링크 추가
  - "전체 복사" 버튼 (카톡/이메일 전달용)
- **기대효과**: 브리핑의 "읽는 맛"과 신뢰감 향상 + 카톡 공유 가능
- **상세**: `docs/HANDOFF-design-polish.md`

#### [B] 본문 크롤링 개선 — 현재 0%

- **현상**: Google News URL 디코딩 전부 실패 → Top 5가 스니펫(1~2줄)만으로 분석
- **접근**:
  1. `requests.head(allow_redirects=True)`로 redirect 추적 (빠르고 가벼움)
  2. `googlenewsdecoder` 최신 버전 확인/교체
  3. 필요 시 Playwright 폴백
- **기대효과**: 크롤링률 0% → 50~90%, Top 5 분석 깊이 향상

#### [D] 카테고리 요약 심화

- **현상**: 카테고리별 동향이 headline+fact만. "그래서 뭔 의미?"가 빠져 있음
- **수정**: 기사 3건 이상인 카테고리에만 impact 필드 추가
- **비용 영향**: Stage 3 토큰 ~20% 증가 ($0.008/일, 무시할 수준)
- **기대효과**: 카테고리 섹션 정보 밀도 향상

---

### 3차: 평가 후 결정

#### [E] 수집 튜닝 Phase B — Haiku LLM 필터

- 과제 A 적용 후 결과 데이터를 보고 필요 여부 판단

- 과제 B(크롤링 개선) 후 본문 3,000자 입력 상태에서 A/B 테스트
- 비용의 50%를 차지하는 핵심 구간 → 품질 확인 필수
