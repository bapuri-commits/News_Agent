| 포맷 선호 | 리스트 | 실제 대화에서 **표 + 비교분석** 더 빈출 | **차이 발견** |

### 4.1 주목할 차이점

1. **포맷 선호 불일치**: 설문에서 "리스트"를 선택했으나, 실제 ChatGPT 사용 패턴에서는 "표"와 "비교분석"을 더 자주 요청. → 브리핑에서 둘 다 제공하는 것이 안전.

2. **도시개발/스마트시티**: 설문에서 3(필수)으로 표시했으나 export에서는 네옴시티 1건뿐. → 최근 관심사이거나, ChatGPT보다 다른 채널에서 정보를 얻고 있을 가능성.

3. **ESG/탄소규제**: 설문에서 비즈니스 테마로 3(필수) 표시했으나, export에서는 간접 언급만. → 규제 리스크 관점에서 중요하다고 인식하지만, 적극적으로 검색하지는 않는 패턴.

4. **뉴스 브리핑 직접 요청 발견**: "삼성 OpenAI LOI" 대화에서 "Send me semiconductor business news at 8am every day"를 실제로 요청한 기록 발견. 이 프로젝트의 존재 이유를 직접 뒷받침하는 증거.

## 7. 다음 단계: Phase 4 (Profile Builder)

Phase 4에서는 위 28개 비즈니스 micro-summary + 설문 결과(father-profile-raw.json)를 병합하여 최종 `stable_profile.json`을 생성한다.

### 병합 시 반영할 사항

1. **설문 3점 항목**: 무조건 top_priorities에 포함
2. **설문 3점 + export 빈출**: 최고 우선순위로 확정 (반도체, DC, EPC/수주, SK그룹)
3. **설문 3점 + export 약함**: top_priorities에 포함하되 open_questions에 "확인 필요" 기록 (도시개발, ESG, 인프라장비)
4. **포맷 선호 차이**: "리스트 + 표 + 비교분석" 모두 지원으로 확장
5. **뉴스 브리핑 니즈 증거**: user_intent에 직접 인용 포함
