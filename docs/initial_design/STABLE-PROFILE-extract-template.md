# Stable Profile 추출 템플릿

> 아버지의 기존 ChatGPT 대화/프롬프트에서 고정 프로필(Stable Profile)을 뽑아내기 위한 입력 폼 + 시스템 프롬프트

---

## 1. 요약 체크리스트 (임원 브리핑 강제 포맷)

아버지의 "운영형 CEO" 관점을 반영한 항목. 요약 모델이 매 기사마다 이 4항목을 강제 출력해야 한다.

| 항목 | 내용 |
|------|------|
| **Fact** (확인된 내용) | 계약/금액/일정/프로젝트 위치/발주처/수주자/공기 |
| **Impact** (회사에 미치는 의미) | 수주믹스, 수익성(원가/공기 리스크), 현금흐름 타이밍 |
| **Risk** (리스크) | 인허가/전력/PF/차환/원가상승/변경관리 |
| **Next Signal** (다음 확인할 것) | 후속 공시, 신용등급 변화, 추가 매각 루머, 프로젝트 진행률 |

## 3. 프로필 추출용 시스템 프롬프트

구현 시 그대로 사용 가능한 형태:

```
너는 사용자의 뉴스 브리핑 에이전트를 위한 "Stable Profile Extractor"다.
입력으로 주어진 사용자의 설명과 (가능하다면) 과거 ChatGPT 대화 일부를 분석해,
장기적으로 유지되는 선호/관심/금기/브리핑 포맷을 구조화된 프로필로 추출한다.

중요 규칙:
- '일시적 기분/단기 관심'은 Stable로 저장하지 말고 Conversation Hint로 분리하라.
- 사실과 추정을 분리하라.
- 사용자의 편향을 강화하지 말고, 소스 다양성/관점 균형이 유지되도록
  프로필에 "가드레일"을 포함하라.
- 민감 정보(개인정보, 내부기밀로 보이는 내용)는 저장하지 말고 경고 플래그로 표시하라.
- 출력은 아래 JSON 스키마 형태로만 작성하라.
```

## 4. 출력 JSON 스키마

```json
{
  "user_intent": "...",

  "must_include_triggers": ["...", "..."],
  "avoid": ["...", "..."],
  "preferred_format": {
    "reading_time": "5min+20min",
    "sections": ["Top5", "ByCategory", "Risks", "NextSignals", "SourceDiversity"],
    "style": "executive_brief",
    "detail_level": "high_fact_density"
  },
  "source_preferences": {
    "preferred_groups": ["..."],
    "avoid_groups": ["..."],
    "paywalled_ok": true
  },
  "schedule": {
    "timezone": "Asia/Seoul",
    "daily_time": "HH:MM",
    "weekends": true
  },
  "conversation_hint_policy": {
    "lookback_days": 7,
    "max_weight_percent": 10
  },
  "risk_guardrails": {
    "diversity_enforced": true,
    "require_links": true,
    "separate_fact_inference": true
  },
  "open_questions": ["결정이 필요한 항목 3~8개"]
}
```
